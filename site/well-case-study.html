Chapter One

The Problem

Oklahoma's oil- and gas-well completions are documented on "1002A" forms. Roughly 571,446 of them survive only as scanned PDFs spanning 113 years of changing layouts — typed, handwritten, stamped, skewed, faded. Buried in each is the well's location: a county, a Section–Township–Range legal description, and on most forms a hand-drawn dot on a "spot well correctly" grid.

At even two minutes per record, transcribing the archive by hand is ~9 person-years. Identifying orphaned, abandoned, and idle wells for carbon-management planning needs those locations as GIS-ready coordinates — so the goal: an automated, validated pipeline with human verification built in.

Chapter Two

The Chronicle of the Build

Nine phases, fifty-nine days, one design law.

  1. May 15–23

    Foundation

    Modular pipeline born: ZIP-native reading, per-record crash recovery, Vision OCR, six-method OpenCV grid detection. First large benchmark on 4,607 records — grid 100%, county 99%, dot 89%.

  2. May 23–31

    Coordinate Resolution

    A 4.56M-cell PLSS PostgreSQL database on AWS RDS; a multi-pass resolver with bilinear interpolation of the hand-drawn dot; portability hardening across the codebase.

  3. May–June

    U-Net Dot Detector

    643 hand-labeled dots train a 192×192 U-Net that replaces brittle classical CV — with regression-gated retraining that can't silently degrade accuracy.

  4. Jun 8–12

    Hardening Sprints

    Every silent crash root-caused to 7.4 GB of RAM. Sharded status saves, self-healing watchdogs, run-scoped resume — startup drops 12.6s → 2.1s.

  5. Jun 10–13

    Human-in-the-Loop

    2,781 hand-reviewed forms become per-collection search envelopes. A public Verify/Fix panel on the map feeds a fully autonomous corrections loop: GitHub issue → Action → RDS re-resolution → republish. No human intervention.

  6. Jun 13–16

    Data Integrity

    Seven-way reconciliation — disk ⇄ ZIPs ⇄ index ⇄ tracker ⇄ coordinates ⇄ S3 ⇄ site — deduplicated to one row per record, with non-destructive merge guards after two near-miss data-loss incidents.

  7. Jun 16

    The Breakthrough

    Modern digital-native records carry typed data: county decodes deterministically from the API number, ~31% print exact coordinates. The map jumps 7,163 → 47,253 wells in one day — at $0.

  8. Jun 16–20

    Precision & Scale

    12,662 wells upgraded to exact printed coordinates; crash-proof chunk-chain campaigns; AWS Batch cloud run designed with a 1,000-vCPU quota approved; AAPG Bulletin manuscript drafted.

  9. Jul 12

    Today

    51,559 wells live across all 77 counties — each pin linked to its original scan — with ~470K records awaiting the cloud run.

Chapter Three

Architecture

Five extraction paths, one resumable ledger, everything falls back.

scanned PDF (ZIP / S3)
  └─ PDFDocumentManager (cached page renders, 2× resolution)
       ├─ latlong/   printed coordinates (decimal + DMS) — modern forms
       ├─ grid/      anchor phrase → measured-envelope crop → CV ensemble
       │               └─ era-aware form classifier
       ├─ location/  STR extraction (per-era recipes → zones → full page)
       ├─ county/    structural anchors → fuzzy match → Gemini fallback
       └─ dot/       U-Net segmentation → PLSS quarter-quarter label
  └─ processing_status.csv   (44-column resumable tracker)
  └─ coord/plss_resolver     (RDS, 5-pass, county-pinned directions)
  └─ build_map_data.py       → GitHub Pages interactive map
The design law, held throughout: hints and priors bound and order the search — they never terminate it. Every stage has a full-page fallback; suspect detections are flagged, never silently trusted. That's why a century of layout drift degraded results gracefully instead of breaking them.

Chapter Four

Key Findings

What half a million documents teach you.

Route by format era

The archive splits into two regimes: pre-2001 scans need CV + paid OCR; post-2010 digital forms carry typed text. Routing by era turned the worst collections (7% grid success) into the best (88–98% mapped, free).

Free OCR fails — and the reason matters

Tesseract scored 0% location agreement on old scans; embedded PDF text 0% on county. Flat text loses page geometry — the extraction problem is spatial, not typographic. Paid document-OCR earns its cost.

The section number is the ceiling

Township and range anchor themselves with direction letters; the section is a bare number whose label OCR drops. Only 44% of legacy records parse a complete STR — the improvement agenda is set by one number.

Ground truth beats heuristics

Mining the PLSS database showed direction is deterministic from county alone in ~59/77 counties — wired into the resolver, and the same insight caught a mirror bug that had flipped ~2,000 wells across the state.

A resolver is a decision tree, not a lookup

Seven ordered strategies run cheapest-first over 4.56M PLSS cells, with a 4-priority direction cascade, soft-overrides of contradicted OCR, batch prefetch (3,500 queries → 2), and section-centroid as an explicitly-labeled last resort.

Constraints shape the engineering

A 7.4 GB laptop OOM-killed every resident orchestrator. The surviving pattern — fresh-process 50-record chunks, idempotent via the tracker — ran 35,845-record campaigns crash-free at ~1,000 records/hour.

Chapter Five

Humans Where They're Irreplaceable

  • 2,781 hand-reviewed forms → measured per-collection search envelopes — the single biggest accuracy lever (one collection's grid success: 48% → ~100%).
  • Predict-then-correct labeling — the model proposes, a human accepts or corrects; 643 dot labels feed retraining that must beat the old model to ship.
  • Public corrections loop — any viewer can compare a pin to its original PDF and submit a fix; a GitHub Action validates, re-resolves, and republishes autonomously.
  • Failure transparency — per-stage failure CSVs regenerate on every publish, for human eyes.

Chapter Six

Scars & Lessons

Every earthbender learns by taking a few hits.

  1. Two near-miss data losses — a script truncated the 514K-row tracker to 3,334 rows; another overwrote the coordinates file. Both recovered from backups, both answered with structural guards. At half-million-record scale, merge tooling must be non-destructive by construction.
  2. A cloud run would have shipped garbage — the container defaulted to the OCR engine proven inaccurate. Caught in a pre-launch scrutiny pass. Audit runtime defaults against experimental findings before scaling.
  3. 148 counties in a 77-county state — unnormalized strings split entities on the public map. Normalize at the publishing boundary, not just extraction.
  4. Memory-kill forensics — silent Windows deaths traced through event logs to RAM exhaustion. The fix was architectural, not a patch.
  5. Honest negative results saved money — the Tesseract and embedded-text pilots returned "no" in hours and prevented a ~$1,500 mistake. Cheap experiments before expensive convictions.

Epilogue

The Six Teachings

  1. Route by format era, then optimize per route — the decision worth the most wells per dollar.
  2. Spatial context is the moat: document understanding is about where text sits, not just what it says.
  3. Hints bound, never gate — fallback-everywhere survives a century of drift.
  4. Idempotence + fresh-process chunking turns unreliable hardware into a reliable batch system.
  5. Put humans where they're irreplaceable; automate the rest — 2,781 reviewed forms leveraged into half a million records.
  6. Reconcile everything to one ledger, or drift will quietly eat correctness.
